I wanted my site to be fast and almost zero-maintenance. The approach I landed on: a statically exported Next.js app where all content lives in Markdown files.

<img src="/projects/portfolio.png" alt="Minimal personal site example" className="mx-auto block w-full max-w-[36rem] rounded-lg" />

## The content model

There are exactly two kinds of content, and both are just files on disk:

- **Posts** live in `content/blog`
- **Projects** live in `content/projects`

Each file starts with a small frontmatter block — title, date, summary — followed by the body. At build time the site reads the folder, parses the frontmatter, and generates a page for each file.

## Projects that link out

Not every project deserves a full writeup. Some are just a link to a repo or a live demo. So a project's frontmatter can include an optional `externalUrl`:

```md
---
title: "Some Tool"
date: "2026-01-01"
externalUrl: "https://github.com/you/some-tool"
---
```

When that field is present, the card links straight out. When it's absent, the card links to a rendered article. Same system, two behaviors.

## Why static

No server means nothing to patch, nothing to pay for, and nothing to fall over at 3am. It deploys to GitHub Pages as plain HTML and loads instantly.
